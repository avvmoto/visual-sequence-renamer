# Project Specification: Smart File Renamer

## 1. 概要
ユーザーが直感的にファイルを並べ替え、その順番通りに連番でリネームできるデスクトップアプリケーション。
macOSとWindowsの両方で動作するクロスプラットフォーム対応。

## 2. 技術スタック
- **Language:** Python 3.10+
- **GUI Framework:** PySide6 (Qt6)
- **Image Handling:** Pillow (サムネイル生成用)
- **Environment:** uv (Package Manager)
- **Lint/Format:** Ruff
- **Path Handling:** pathlib (OS依存のパス問題を回避)

## 3. 主要機能 (MVP)

### A. ファイル入力と表示
- **ドラッグ&ドロップ:** アプリ画面へのファイルドロップでリストに追加。
- **サムネイル表示:** - 画像ファイルはアスペクト比を維持して大きく表示（初期値: 120x120px程度）。
    - 画像以外はファイル形式に応じた汎用アイコンを表示。
- **リスト管理:**
    - ドロップされたファイルを一覧表示。
    - アイテムを選択して `Delete` キーでリストから削除可能。
    - 「すべてクリア」ボタンでリストをリセット。

### B. 並べ替え機能
- **GUIによる並べ替え:** リスト内のアイテムをマウスでドラッグして、直感的に順番を入れ替え可能。
- **リアルタイム反映:** 並べ替えた順番がそのままリネーム後の連番（001, 002...）に対応する。

### C. リネームロジック
- **命名規則:** `[連番]_[元のファイル名].[拡張子]` (例: `001_photo.jpg`)。
- **実行:** 「リネーム実行」ボタン押下で処理開始。
- **出力先:** - デフォルト: 最初にドロップしたファイルのディレクトリ内に `renamed_[YYYYMMDD_HHMM]` フォルダを自動生成。
    - 手動設定: ユーザーが任意の出力フォルダを選択可能。
- **完了通知:** 処理完了後、メッセージを表示し、出力先フォルダをエクスプローラー/Finderで自動的に開く。

## 4. 非機能要件・UX
- **OS対応:** WindowsとmacOSでUIの挙動（メニュー、ショートカット）を最適化。
- **非同期処理:** 大量の画像ドロップ時にUIがフリーズしないよう、サムネイル生成とリネーム実行は `QThread` を使用。
- **堅牢性:** ファイル書き込み権限エラーや、同名ファイル存在時の例外処理を実装。

## 5. テスト・動作確認 (Smoke Test)
- **起動テスト:** アプリが正常に立ち上がること。
- **ハッピーパステスト:** - ファイルドロップ → 並べ替え → 実行 → 出力フォルダにファイルが存在する、という一連の流れを確認する `pytest-qt` ベースのスモークテストを作成。
