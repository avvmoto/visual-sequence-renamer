# Project Specification: Smart File Renamer

本ドキュメントは **現在の実装の正** とする。コードと乖離した場合は本ファイルを更新する。

## 1. 概要

同一フォルダ内のファイルを一覧し、表示順どおりに連番付きファイル名へ**その場リネーム**するデスクトップアプリケーション。macOS と Windows の両方を想定したクロスプラットフォーム対応。

## 2. 技術スタック

| 項目 | 採用 |
|------|------|
| Language | Python 3.10+ |
| GUI | PySide6 (Qt6) |
| 画像 | Pillow（サムネイル） |
| パッケージ管理 | uv |
| Lint / Format | Ruff |
| 型検査 | Mypy |
| テスト | Pytest（`pytest-qt` を dev 依存に含む） |
| パス | `pathlib.Path` を用いる |
| Windows 配布用ビルド | PyInstaller（CI のみ dev で使用） |

## 4. 主要機能

### 4.1 登録フォルダの制限

- **最初に追加したファイルと同じ親フォルダ内のファイルのみ**リストに登録できる。
- 別フォルダからのドロップはスキップし、警告を表示する。
- リストが空になると登録フォルダの制約はリセットされる。

### 4.2 ファイル入力と表示

- **ドラッグ＆ドロップ:** Finder / エクスプローラーからファイルをドロップして追加。挿入位置はドロップ位置に応じる（既存アイテム間への挿入）。
- **重複:** 同一ファイル（解決済みパスが同一）は追加しない。
- **サムネイル:** 画像はアスペクト比を維持したサムネイル（アイコンサイズ約 120px）。非画像は OS / Qt のファイルアイコン。
- **ラベル:** カスタムデリゲートで「アイコン上・ファイル名は下」。長い名前はアイコン幅で省略（`...`）。ツールチップにフル名。
- **リスト表示:** `QListWidget` を ListMode＋左から右へのフロー＋折り返しでグリッド状に配置（内部ドラッグ並べ替えとの兼ね合い）。
- **リストのクリア:** ツールバーの **「クリア」** で一覧を空にし、追加順カウンタとソート選択（追加順）をリセット。
- **個別削除（Delete キー）:** 現状 **未実装**。

### 4.3 ツールバー

- **クリア:** 上記。
- **ソート項目（QComboBox）:**  
  - 追加順（内部の追加連番）  
  - ファイル名  
  - 作成日時  
  - 更新日時  
  - 追加順（カスタム）— 主にリスト内ドラッグ直後に自動選択  
- **昇順 / 降順（ボタン）:** `↑` / `↓` で切り替え。「追加順（カスタム）」選択中は無効。
- ソートは **表示順のみ**変更し、ディスク上のファイルは動かさない。

### 4.4 メタデータとソート

- 追加時に `pathlib` / `stat` により **作成時刻・更新時刻** を取得し、`QListWidgetItem` のカスタム `DataRole` に保持（後続ソート用）。
- 作成時刻は OS により取得できない場合があり、そのときは代替値を用いる（実装依存）。
- ソートは `O(n log n)`。ソート処理中は一時的にリストが空になる副作用を避けるため、登録フォルダリセットと競合しないよう制御している。
- **リスト内で手動ドラッグして並べ替えた直後**、ソート項目は自動的に **「追加順（カスタム）」** に切り替わる（このモードでは自動ソートしない）。

### 4.5 並べ替えとリネーム順

- リスト内ドラッグで順序を変更可能（複数選択の移動に対応）。
- **画面上の上から順（行順）がリネーム時の連番順**に対応する。

### 4.6 リネーム

- **命名規則:** `[連番]_[ベース名].[拡張子]`（例: `001_photo.jpg`）。連番桁数は件数に応じ最低 3 桁。
- **実行場所:** 登録フォルダ内で **リネームのみ**（別フォルダへのコピーや `renamed_日時` フォルダは**行わない**）。
- **実装:** 一時 `.tmp` 名を挟んで相互上書きを避ける。
- **オプション（チェックボックス、既定 ON）:** ファイル名 stem 先頭の **`数字3桁以上 + アンダースコア`** を繰り返し取り除いてから連番を付け直す（同一フォルダで繰り返しリネームしたときの連番の積み上がりを抑える）。
- **実行:** 「リネーム実行」。処理は **`QThread`** 上で実行し UI をブロックしない。
- **完了:** メッセージ表示のうえ、該当フォルダを Finder / エクスプローラーで開く（失敗時はパスを表示）。
- **エラー:** 権限・同名衝突などはメッセージで通知。
- **元に戻す:** 直近のリネームを **`QThread`** で逆リネーム。成功後は一覧のパス・表示を更新。リストを編集すると元に戻しは無効化される場合がある（実装に準拠）。

### 4.7 エントリ・終了

- `uv run renamer` または `python -m app.main`。
- ターミナル起動時は **SIGINT（Ctrl+C）** および **KeyboardInterrupt** でイベントループを終了してアプリを終了できるよう `main` で処理する。

## 5. 非機能要件・UX

- **クロスプラットフォーム:** Windows / macOS を想定。外部ファイル D&D は OS 差のある挙動に配慮した実装。
- **非同期:** リネーム・元に戻しは `QThread`。サムネイルは追加時にメインスレッドで生成（大量ファイル時は将来ワーカー化の余地あり）。
- **堅牢性:** ファイル操作・画像読み込みは例外を捕捉しユーザーに通知する方針。

## 6. CI / 配布

- **GitHub Actions**（`.github/workflows/build.yml`）  
  - **トリガー:** `main` ブランチへの `push` のみ。  
  - **ランナー:** `windows-latest`。  
  - **uv:** `astral-sh/setup-uv`、**キャッシュ有効**。  
  - **ビルド:** `uv sync --all-groups` のうえ **PyInstaller**（`--windowed`、`--collect-all PySide6` / `shiboken6`、エントリ `src/app/main.py`）。  
  - **成果物:** `dist/` を **GitHub Artifacts**（`renamer-windows`、保持 30 日）にアップロード。  
- **自動 GitHub Release へのアップロード:** 現状 **未設定**（必要なら別ワークフローで拡張）。

## 7. テスト

- **test_smoke.py:** パッケージ・`main` のインポート等の最小スモーク。
- **test_rename.py:** 連番名生成・同一フォルダリネーム・アンドゥ・プレフィックス除去のロジック。
- **pytest-qt による E2E スモーク:** 仕様上は推奨だが、**現状は未整備**（拡張余地として記載）。

## 8. 旧仕様からの変更メモ（参照用）

以下は **もう採用していない** 方針である。

- `renamed_YYYYMMDD_HHMM` サブフォルダへのコピー出力。
- 任意の出力先フォルダの手動選択。
- 登録フォルダをまたいだファイルの追加。
